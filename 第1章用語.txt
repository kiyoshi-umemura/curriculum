■ ネットワーク : パソコンやネットワーク機器といった複数の要素がケーブルなどで接続され、相互に通信できる状態になっているもの。

・LAN: 建物内、敷地内などの狭い範囲のネットワーク。
	有線LAN : LANケーブルを使った接続方法。
	無線LAN : LANケーブルを使わず無線通信での接続方法。

・WAN: 地理的に離れたLANとLANを結びつけることで構築する広い範囲のネットワーク。
	インターネット: プロバイダと呼ばれる通信事業者を介して構築される世界規模のネットワーク。  
 

■ ネットワーク構成　：LAN,WAN,インターネットに分類できる。

・ノード : ネットワークにおける機器のこと。コンピュータ、プリンタ、ルータなど。
・リンク: ノードとノードの間のデータの通り道のこと。ケーブルや電話など。
・トポロジ : 端末やネットワーク機器をどのように接続しているのか、その接続形態。
	バス型トポロジ : バスと呼ばれる一本の同軸ケーブルに複数のノードを接続するトロポジ。バスの終端には電気信号の反射 
                   によるノイズを防ぐターミネータ（終端抵抗）が取り付けられている。１箇所でも断線すると全体が機能し
　　　　　　　　　　  なくなる。
	スター型トポロジ : 1つの集線装置を中心にノードを接続するトロポジ。扱いやすく耐障害や拡張性に優れるため、現在の
                    LAN構築で最も一般的に使用されるトロポジである。
	メッシュ型トポロジ : 複数のノードを網目状に接続するトロポジ。コストはかかるが高い冗長性を持つ。
・帯域 : 通じてなど用いる一番低い周波数と一番高い周波数の間の、周波数の範囲。
	bps : 通信回線などのデータ伝送速度の単位。
・輻輳 : 大量のデータ通信などが原因でネットワークが混雑し、データ通信が困難になること。
・コリジョン : データの衝突。
・ドメイン : インターネット上の住所。Webサイトがどこにあるかを判別する情報として利用する。
・ポート : 機器やソフトウェアが外部の別の主体と接続・通信するための末端部分のこと。
 

■ モデル

・プロトコル : 通信におけるルールのこと。
・OSI参照モデル : 異なる機種間でも通信できる世界標準のプロトコルスタック。
	ヘッダ : 各階層でデータを送信するために必要な情報。
	カプセル化 : 上の階層の通信データをそのままデータとし、新たなヘッダを付加すること。
	非カプセル化 : ヘッダを外してデータ部分を取り出すこと。
	PDU : 各層で扱う通信データの単位。
	　・MTU : 1回の通信で送信することができるパケット最大サイズのこと。
	　・フレーム : インターネット層から渡されたパケットにイーサネットヘッダとトレーラと呼ばれる誤り検出のための情報を後ろに付加し、この付加されたデータのこと。
	　・ パケット : IPヘッダが付加されたデータ。
	　 ・セグメント : TCPヘッダが付加されたもの。
・TCP/IPモデル : 現在インターネットで利用されている通信プロトコル群のこと。
 

■ 物理層　：データを電気信号に変換するといったことや、物理的な接続に関して規定しています。

・ビット : コンピュータが扱うデータの最小単位。
・ツイストペアケーブル : 銅線を2本ずつペアにしてより合わせたケーブル。
	UTP : シールドなしツイストペアケーブル。
	STP : ノイズに強いシールド付きのツイストペアケーブル。
	カテゴリー : 対応する規格。
	コネクタ : 電線と電線または電線と電気器具を接続導通する部品。
・ストレートケーブル : 8本の銅線を全く同じ並びでコネクタに接続しているケーブル。ケーブルの両端でピンの並びが同じとなっている。
・クロスケーブル : 8本の銅線をクロスさせて接続しているケーブル。ケーブルの両端でピンの並びが異なっている。
・光ファイバケーブル : 電気信号ではなく、光によって通信するケーブル。中心部のコアとその周りのクラッドをさらに皮膜で覆った構
                                 成。・メリット : 回線速度が速く、安定している・デメリット : 取扱が難しい
・ハブ : 通信データを単なる電気信号としてしか扱わない物理層で働く装置の１つ。集線装置としての機能+リピーター機能を有す　　
            る。特定の宛先を選んで送るといった制御機能はない。
	
　　　半二重通信 : データの送信と受信を同時に行わず、送信側と受信側が交互にデータを送信することで通信を行う方式。

■ データリンク層　:　直接接続されたノード間の通信に関して規定している。

・イーサネット
	イーサネット  : LANや一部のWANにおいてリンク層(データリンク層と物理層）で利用されるプロトコル。
	ファストイーサネット : 有線の構内ネットワーク（LAN )の標準の一つであるイーサネット。
	ギガビットイーサネット : 最大通信速度が１ギガビット毎秒(1Gbps)である仕様のイーサネットの規格。
・イーサネットヘッダ : OSI参照モデルの物理層およびデータリンク層の有線LAN通信において代表的な規格。
・トレーラ : データ区切って転送するときに、その区切られたデータひとつひとつに付加される制御情報。
・CSMA/CD : データの衝突を避けるための仕組みと。万が一衝突が発生した場合にデータを再送する仕組みを兼ね備えた通信制御方法。
・MACアドレス : ネットワークインターフェースを識別するために使用される識別子。
・スイッチ : ネットワークを構成する上で重要な機器で、集線装置であり、接続したコンピュータなどの機器同士の相互通信を可能にする。

	MACアドレステーブル :ネットワークスイッチなど内部的に管理している情報で、どのポートにどのMACアドレスをもつ機器が接続されているか記録したもの。
	フラッディング : ネットワーク機器の処理能力を超えるデータ量が発生し、ネットワークシステムの動作に支障がでる状態のこと。
	全二重通信 : データの送信と受信を送信側と受信側が同時に行うことができる方式。
・VLAN : スイッチの技術の1つで、スイッチの内部でネットワークを仮想的に分ける技術のことを指す。
	タグVLAN : VLAN（仮想LAN）の方式の一つで、ネットワークを流れる個々のフレームに転送先のグループの識別番号を付
                               与する方式。
	ポートベースVLAN : VLAN（仮想LAN/バーチャルLAN）の方式の一つで、一台のネットワークスイッチ（スイッチングハブ）の
                     差込口（ポート）を単位に、物理的な回線でグループを構成する方式。VLANのうち最も単純な手法
                                          である。
	　・アクセスポート : VLAN（仮想LAN）が設定されたLANスイッチなどで、一つのVLANのみに所属するポートのこと。主に端
                                          末からの接続を受け付けるポートに指定する。
 
	　・トランクポート : VLAN（仮想LAN）が設定されたLANスイッチなどで、複数のVLANに所属し、それぞれのデータを混在さ
                                         せて伝送するポートのこと。主にスイッチ間接続で用いられる。
	DTP : ネゴシエーションを行うプロトコル。
	デフォルトVLAN　：　VLANを設定したネットワークスイッチで、VLANの指定が無いポートやフレームの通信に用いられるVLAN
	ネイティブVLAN　：　タグ付きポートで受信したタグなしパケットの所属先VLANのこと。
・ブロードキャストストーム : 不特定多数に対して同じデータを送信する方式。１対多数での通信になります。
・スパニングツリープロトコル　: ブロードキャストストームを防ぐためのもの。
・リンクアグリゲーション　：　複数の物理リンクを１つの論理的なリンクとみなし、複数のリンクを効率よく利用する技術。
・EtherChannel　:　複数の物理リンクを１つの論理的なリンクとみなし、複数のリンクを効率よく利用する技術。
